This page will test the getJavaInfo.jar applet in this folder to see if it works and is compatible with the PluginDetect Library...

In order for this test to work, you will need the following:
1) Make sure that Java is installed and enabled in your browser.
2) Your browser and your version of Java need to communicate via the Javascript/Java bridge. Certain combinations of browsers and Java versions may not permit this communication.
3) Make sure that your getJavaInfo.jar applet is in this folder.
4) This test only needs to work on one browser and one version of Java in order for you to verify the getJavaInfo.jar applet.
5) If your browser has deactivated the Java plugin due to security reasons, then you will have to activate it (ie. give security approval to run the plugin).
6) For Java 7 update 51 and higher, unsigned applets no longer run by default. If the applet you are using is unsigned, then you may have to go into your Java Control Panel and add this web page (or just the domain) to the Exception Site List. As an alternative, you could use a signed applet - but the cost for the certificate is high.
7) This web page should probably be served by a WEB SERVER to your browser. The reason is that some browsers/Java versions/Java security settings may require a WEB SERVER to be used in order for Java to run properly in a web page. (ie. your browser accesses the web page via a server, instead of accessing the web page directly from your local disk drive)
